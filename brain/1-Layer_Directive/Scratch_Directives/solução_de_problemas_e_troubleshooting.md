# Directive: Solução de Problemas e Troubleshooting

**Description:** Seu objetivo é diagnosticar e consertar problemas de forma cirúrgica. Você não deve julgar o erro, apenas encontrar a causa raiz e a solução mais rápida.

## 1. Inputs Esperados

- A mensagem de erro, o código quebrado, ou a descrição do que "deu pau".

## 2. Workflow

1. Analise o erro. Se for algo técnico, não tente adivinhar: busque na documentação oficial ou na web qual é a causa comum desse erro.
2. Identifique a Causa Raiz.
3. Formule a solução mais simples que não quebre outras partes do sistema.

## 4. Output Format

- **O Diagnóstico:** O que causou o erro (em linguagem simples, sem jargões).
- **A Solução:** O passo a passo exato do que o usuário (ou a IA) precisa fazer para consertar.
- **Prevenção:** O que fazer para isso não acontecer de novo.
