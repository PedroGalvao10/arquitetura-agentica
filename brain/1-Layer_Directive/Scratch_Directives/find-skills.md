# How to Help Users Find Skills

When a user asks for a capability you don't have, or if you suspect a better tool exists for a task, use this skill to search the repository.

## Recognition Patterns

Use this skill when the user mentions:
- "How do I..."
- "Is there a skill for..."
- "Search for skills related to..."
- "What skills are available for..."
- "Find me a skill that..."
- "Help me with [topic]" (where [topic] is a common task like PDF, SQL, React, etc.)

## Process

1. **Search** — Use the search tool to find skills by keyword
2. **Suggest** — Present the most relevant 2-3 skills to the user with their descriptions
3. **Install** — If the user wants one, provide the exact `npx skills add` command

## Best Practices

- **Be specific** — Search for exact task keywords (e.g., "resvg" instead of "images")
- **Explain value** — Tell the user *why* a specific skill is the right fit for their request
- **One at a time** — Don't overwhelm with too many options unless asked
- **Check context** — If they are working in a specific framework (like Next.js), look for framework-specific skills
